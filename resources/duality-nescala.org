* What is duality?

  :speakernotes:
A little CT to start with: for any category /C/, there is an opposite (or dual) category C^op with all the arrows reversed (but the same objects). It follows from reversing the arrows again, that the dual category of C^op is C.

So far, so easy, right? Thatâ€™s fundamentally all there is to it, but there are some complications
  :END:

- C
- C â†¦ C^op
- (C^op)^op = C

- C and C^op have the same objects
- every arrow /A â†’ B/ in C has a corresponding arrow /B â†’ A/ in C^op

:speakernotes: 
So, we declare a new data type representing the fixed-point of functors. We donâ€™t need to care about the specific data type so much, other than to note that it is its own dual, and that the constructor for it is dual to its sole accessor. Then we define a function that uses this type, wrapping it in a bit of magic that says â€œalso define the dual of this function, and give it the name ~ana~.â€ The generated dual looks like:
:END:            

#+begin_src haskell
ana :: (a -> f a) -> a -> Fix f
ana f = Fix . fmap (ana f) . f
#+end_src


:speakernotes: 
So, letâ€™s walk through this a bit â€¦ the first argument is a function (and the dual of a function just inverts the arrows, as weâ€™ve seen). And then we hit the same issue we did with ~Functor~ â€“ rather than this being a function of two arguments, itâ€™s really a function that takes a function and returns a function. So, we flip the arrow of the returned function as well. So, we now have the correct type for ~ana~.

Now, the value â€¦ first, the dual of ~f . g~ is ~g . f~, so all the compositions should be flipped, which means we compare ~project~ to ~Fix~, ~fmap~ to ~fmap~, and ~f~ to ~f~. Weâ€™ve just declared that ~Fix~ and ~project~ are duals, so that works. Variables are sort of self-dual (their types have been dualized, so it really represents a different expression), so ~f~ works as well. Finally, we have ~fmap~. We know that that is self-dual already, but we have to take the dual of the function we pass to it, and as weâ€™ve just defined here, ~ana~ is (going to be) the dual of ~cata~, so we swap that in there, and great â€“ it all checks out!
:END:
